## 信息架构 / 页面流程

```
首页 (Homepage)
  │  ├─ 首页1 / 首页2 = 同一页，照片帧序列动画的两帧（循环）
  │  └─ 触发：点击向下箭头 / 向下滚动
  ▼  （过渡 A：交叉重叠，头像作锚点）
进入网页 (Landing / 作品总览)
  │  ├─ 左上：店主头像（从首页飞入）+ RPG 绿色对话框
  │  ├─ Main 主推作品大展示位
  │  └─ 3D(ほか) / 2D(ほか) 缩略图网格
  │  触发：点击某个作品缩略图（或向下滚动）
  ▼  （过渡 B：交叉重叠，被点的缩略图作锚点）
进入作品 (作品详情)  ※ 店主头像不出现在本页
     ├─ 左大白框：主展示区（承接飞入的缩略图，静音 mp4 循环）
     ├─ 右上橙框：细节展示视频 mp4（点击弹出放大）
     ├─ 右下橙框：主模型旋转帧序列 turntable
     ├─ 右侧绿字说明 + text sample 链接
     ├─ modlesDownloadLink 模型下载
     ├─ 返回键 → 倒放进入动画，回到「进入网页」网格（自由挑作品）
     └─ 页面最底部点击 → 卷轴式进入「下一个（我安排的顺序）」作品

[左下角线框 = 音乐播放器彩蛋，跨页面常驻，状态不随过渡重置]
```

## 命名规范（用户已定，必须遵守）

- **驼峰截断**：命名用驼峰式（camelCase），过长单词截断缩写
- **素材文件名模式**：`类型前缀_(项目名)(序号).扩展名`
  - 图片：`Image_(项目名)(序号).png` → 占位：`Image_Sample1.png`
  - 视频：`video_(项目名)(序号).mp4` → 占位：`video_Sample1.mp4`
  - ✅ 前缀拼写已由用户确认修正：**`video_`**（原 "vedio" 为误拼，一律用 `video_`）
- **markdown 说明内嵌的图片也走同一套 `Image_(项目名)(序号).png` 编号**，不单独命名——序号在该项目内全局连续，方便后续微调视觉细节
- **占位项目名**：`Sample`（用户笔误写过 "Samle"，以占位文件名 `Image_Sample1.png` 为准）
- **序号 = 展示优先级**：
  - 序号越小越靠前，**1 = 优先/主展示**
  - **若该作品只有序号 1（没有 2、3……）**：版式退化为「素材 1 + 旁边说明文字」的形式
  - → 作品详情页布局必须**按素材数量自适应**，不能写死框数
- **说明文章**：用 **markdown**（或同类笔记格式）编写，支持「文字（图片）文字」交错的内容箱子——说明是可穿插图片的富文本流，渲染时按块解析

## 商品数据模型（扩展性核心，必须遵守）

**这是一家"店"，商品（作品）要能像真实商店一样随时上新 / 改信息。**
所以**全站作品内容必须数据驱动，绝不把单个作品写死在页面里**。

- 作品总览的缩略图网格、每个作品详情页，都**由一份作品数据表渲染生成**
- 新增一个作品 = 往数据表加一条记录，**不需要改页面结构 / 不需要新建 HTML**
- 数据表与素材文件解耦：数据里存路径，素材放资产目录

### 作品数据 schema（字段拟定，可增不可少）

```jsonc
{
  "id": "work_xxx",            // 唯一 ID，过渡动画用它定位网格槽位
  "projectName": "Sample",    // 项目名（驼峰截断），用于拼素材文件名
  "category": "3D",           // "3D" | "2D"，决定进哪个网格区
  "order": 1,                 // 我安排的顺序（全站一条线），底部"下一个"按此递增
  "title": "· (作品名)",       // 作品页左上标题
  "thumb": "asset/...",       // 缩略图（网格里显示，也是过渡 B 的飞入源）
  "images": ["Image_Sample1.png", "Image_Sample2.png"],  // 编号图片，1 = 主展示
  "videos": ["video_Sample1.mp4", "video_Sample2.mp4"],  // 编号视频，1 = 主展示（左大白框），2 = 细节（右上橙框）
  "turntable": ["asset/.../000.png", "..."], // 右下橙框 模型旋转帧序列
  "descriptionMd": "asset/Sample.md",  // 说明文章（markdown，文字/图片交错，文案待补）
  "downloadUrl": ""           // modlesDownloadLink 模型下载链接（【可选】，无则隐藏链接）
}
```

- 字段缺失要优雅降级：没有 `turntable` 就隐藏右下橙框，没有 `downloadUrl` 就隐藏下载链接，等等
- **素材按序号自适应布局**：只有 1 号素材 → 「素材 1 + 旁边说明文字」版式；有 2、3 号 → 细节框依次启用
- `order` 决定"下一个"的串联顺序；`category` 决定缩略图落在 3D 还是 2D 网格
- 网格**每行视觉基准最多 3 列**（4+ 自动换行），**实际框数 = 该分类作品数**，可为 0 / 1 / 2 / N；不补空橙框占位；**作品从架子最右侧起摆、向左增长**（1 个=最右格，2 个=右两格）；分类标题（3D/2D）始终保留

### 说明文字编辑约定（`descriptionMd`）

作品页右侧说明区**完全由项目文件夹内的 markdown 驱动**，`works.js` 的 `descriptionMd` 只登记路径，**不含正文**。

- **编辑源（唯一）**：`portWebsite/asset/works/<项目文件夹>/<说明.md>`
- **本地快捷路径**：根目录 `asset/works/` 已通过目录联接（Junction）指向 `portWebsite/asset/works/`，两处编辑的是**同一份文件**
- **运行时**：`workPage.js` fetch 该 md，按空行分块渲染文字、`{{image:N}}`、`{{row:N,M}}`、`{{video:N}}`、`{{code:path}}`
- **根目录其余副本**：`js/`、`css/`、`index.html` 等仍由 CI（`sync-site-root.yml`）从 `portWebsite/` 自动同步；**改说明 md 可任选联接路径**
- **本地预览**：推荐打开 `portWebsite/index.html`；push 到 main 后根目录与线上同步
- **改标题 / 用时 / 图片列表**：改 `portWebsite/js/data/works.js` 对应条目；**改说明正文**：只改项目 md，通常无需动 `works.js`

## 页面规格

### 首页 Homepage

像一张绿色粗描边会员卡 / 名片。
- **绿框装饰**：`HomepageFrame.png` 用 CSS `border-image` **九宫格（9-slice）** 实现自适应——上下边只拉 X、左右边只拉 Y、四角固定；中间不加 `fill`，透明露出内容。切线与边宽变量见 `css/pages/home.css` 内 **`可调整-边框-大小`**

## 页面间过渡规格

### 通用原则

- 触发：**点击向下箭头 / 向下滚动**
- 模式：**交叉重叠（cross-fade）** —— 旧页元素淡出的同时，新页元素已开始淡入，不排队
- 每段过渡尽量保留一个**连续视觉锚点**衔接两页
- **运动方向语法（全站统一，2026-07 用户修订）**：前进 = 元素**一边 fade out 一边向上滑动消失**；返回/倒放 = true reverse，元素**从上方降回出现**（含位移方向，**不是原地淡入**）。装饰边框类元素例外：**放大 + fade out**（镜头聚焦），不参与上滑

### 过渡 A：首页 → 进入网页

以下同时发生、互相交叠：

| 元素 | 动作 |
|------|------|
| 白色手绘贴纸装饰框（PNG） | **不上滑**：原地**放大 + fade out → 溢出画面消失**（像镜头向中间聚焦推进） |
| 红色爆炸贴纸头像（帧序列） | **缩小 + 直线移动到左上角**，落点 = 进入网页的小头像位；**移动全程帧序列继续播放，不定格** |
| Shop 标题 / 信息块 / 箭头 | 整体**一边 fade out 一边向上滑动消失** |
| **NEW!!** | 独立文字，同样**上滑 + fade out 消失**（不跟随头像） |
| 进入网页全部元素 | **淡出约 0.55s（较快）/ 淡入约 0.9s（较慢）** 交叉重叠渐显，旧页先让路 |

**锚点 = 店主头像**（唯一贯穿两页、不消失的元素）。
**倒放（进入网页 → 首页，↑ 键 / 滚轮上滑 / 顶部向上箭头）**：true reverse —— landing 淡出、首页元素从上方降回 + 淡入、边框从放大态缩回 + 淡入、头像沿直线飞回首页原位。

### 过渡 B：进入网页 → 进入作品

- 旧元素（进入网页，**含店主头像**）淡出，新元素（进入作品）渐显，**交叉重叠**
- **锚点 = 被点击的作品缩略图**：该缩略图**放大并移动到左边的大白框里 → 落位后直接开始播放静音 mp4**
- ⚠️ **头像不是锚点、不携带进作品页**，它随「进入网页」整页一起淡出

## RPG 绿色对话框（进入网页）

店主头像旁的绿色线框 = RPG 风格对话框，逐句推进台词。

**机制：**
- **打字机逐字显示**，节奏偏快，**单句打字动画 ≤ 1 秒**
- **非阻塞**：打字过程中**不锁浏览者其他操作**（可同时滚动 / 点缩略图 / 看 Main 区），对话框自顾自播
- 当前句打完 → 右下小三角（▽）开始**闪烁** = "继续/下一句"提示符
- 推进：点击 / 按键
- **skip 规则**：当前句未打完时点击 → 第一下**瞬间补全当前整句**，**再点一次**才跳下一句（RPG 标准，防误触）
- **首句台词已按参考图登记**（静态展示）：「いらっしゃいませ！ご来店ありがとうございます。」「私のポートフォリオです。」—— 打字机逐句推进状态机仍待 `dialog.js` 接管

## 作品页导航（两条路径，行为不同）

作品页有两种离开方式，对应两种交互逻辑：

| 路径 | 入口 | 交互逻辑 | 去向 |
|------|------|---------|------|
| **返回**（自由挑） | 返回键 | **倒放进入动画**（过渡 B 反向）：当前作品的主展示 mp4 **缩回成缩略图、飞回它在网格里的原槽位**；同时随该页淡出的元素**向上升起重新出现**（遵循运动方向语法） | 回到「进入网页」作品总览网格，用户自己挑下一个感兴趣的 |
| **下一个**（看我安排） | 页面**最底部**点击 | **卷轴式过渡**：当前作品向上卷出，下一个作品卷入 | 按 `order` 递增进入下一个作品 |

**要点：**
- "返回"用倒放，所以飞回的目标槽位 = 当前作品 `id` 在网格中的位置（数据驱动，不能写死）
- `order` 是**全站一条线**（不分 3D/2D，从头串到尾），"下一个"沿这条线递增
- **到队尾时**：「下一个」按钮**变灰禁用（disabled）**，不循环、不跳转
- 两条路径都要保证**左下角音乐播放器不中断、状态保留**
- 卷轴过渡时，新作品的主展示 mp4 进入视口后再开始播，避免一次性预载所有作品视频

## 视频弹出放大（橙框细节视频，非原生全屏）

点击橙框里的 mp4 → 放大查看，但**不是浏览器原生全屏**：

- 视频从原位**放大弹出至屏幕约 70%**，**居中**显示
- 后方背景**变暗**，营造**正片叠底（multiply）**的压暗遮罩感
- **点击视频以外的空白处（遮罩区）** → 视频**缩回原位**、遮罩消失
- 开/合都是从橙框原位 scale 进出（与遮罩淡入淡出同步），不是硬切
- 实现：自定义 lightbox 容器（非 Fullscreen API），遮罩层 `mix-blend-mode: multiply` 或半透明黑

## 技术栈（拟定）

| 职责 | 技术 |
|------|------|
| 页面间过渡 | SPA 式单页 + 区块淡入淡出（保证音乐播放器常驻、头像锚点连续） |
| 作品渲染 | **数据驱动**：一份作品数据表渲染网格 + 详情页，新增作品只改数据 |
| 帧序列播放 | sprite sheet / `<img>` 序列切换 / `<canvas>`（按帧数与体积取舍） |
| 作品 mp4 | `<video>` autoplay + muted + loop + playsinline；橙框视频点击 → 自定义 lightbox 弹出放大（~70% 居中 + multiply 压暗遮罩），点遮罩缩回 |
| 模型 turntable | 帧序列 + 指针拖拽事件（pointermove）控制当前帧 + 闲置自动步进 |
| 返回（倒放） | 复用过渡 B 关键帧，方向取反：主 mp4 缩回缩略图飞回原网格槽位 |
| 下一个（卷轴） | 纵向 scroll/translateY 过渡，按 `order` 串联，视口内才播放新作品 mp4 |
| RPG 对话框 | 打字机状态机（typing / done / advancing）+ 非阻塞渲染 |
| 音乐播放器 | 常驻 `<audio>`，点击线框 toggle，状态跨页保留 |
| 缓动 | 关键帧查表 + smoothstep（复用 frin-site 方案） |
| blobworld 背景 | 渐变 + blob 素材（PNG/SVG），手绘装饰一律 PNG |
