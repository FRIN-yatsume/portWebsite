## 商品数据模型（扩展性核心，必须遵守）

**这是一家"店"，商品（作品）要能像真实商店一样随时上新 / 改信息。**
所以**全站作品内容必须数据驱动，绝不把单个作品写死在页面里**。

- 作品总览的缩略图网格、每个作品详情页，都**由一份作品数据表渲染生成**
- 新增一个作品 = 往数据表加一条记录，**不需要改页面结构 / 不需要新建 HTML**
- 数据表与素材文件解耦：数据里存路径，素材放资产目录

### 作品数据 schema（字段拟定，可增不可少）

```jsonc
{
  "id": "work_xxx",            // 唯一 ID，过渡动画用它定位网格槽位
  "category": "3D",           // "3D" | "2D"，决定进哪个网格区
  "order": 1,                 // 我安排的顺序，底部"下一个"按此递增
  "title": "· (作品名)",       // 作品页左上标题
  "thumb": "asset/...",       // 缩略图（网格里显示，也是过渡 B 的飞入源）
  "mainVideo": "asset/...",   // 左大白框主展示 mp4（静音循环）
  "detailVideo": "asset/...", // 右上橙框细节 mp4（静音循环，可全屏）
  "turntable": ["asset/.../000.png", "..."], // 右下橙框 模型旋转帧序列
  "description": "",          // 右侧绿字说明（文案待补，先留空）
  "downloadUrl": ""           // modlesDownloadLink 模型下载链接（【可选】，无则隐藏链接）
}
```

- 字段缺失要优雅降级：没有 `turntable` 就隐藏右下橙框，没有 `downloadUrl` 就隐藏下载链接，等等
- `order` 决定"下一个"的串联顺序；`category` 决定缩略图落在 3D 还是 2D 网格
- 网格目前每排三个是**视觉示意，不是数量上限**——容器要能容纳更多作品（多排 / 滚动），别写死 6 个

## 页面间过渡规格

### 通用原则

- 触发：**点击向下箭头 / 向下滚动**
- 模式：**交叉重叠（cross-fade）** —— 旧页元素淡出的同时，新页元素已开始淡入，不排队
- 每段过渡尽量保留一个**连续视觉锚点**衔接两页
- **运动方向语法（全站统一）**：前进 = 元素**向下沉没消失**；返回/倒放 = 元素**向上升起出现**（true reverse，含位移方向，**不是原地淡入**）

### 过渡 A：首页 → 进入网页

以下同时发生、互相交叠：

| 元素 | 动作 |
|------|------|
| 白色手绘贴纸装饰框（PNG） | 原地**放大 → 溢出画面消失** |
| 红色爆炸贴纸头像（帧序列） | **缩小 + 直线移动到左上角**，落点 = 进入网页的小头像位；**移动全程帧序列继续播放，不定格** |
| Shop 标题 / 信息块 / 箭头 | 整体**向下淡出消失** |
| **NEW!!** | 独立文字，同样**向下淡出消失**（不跟随头像） |
| 进入网页全部元素 | 在约 **1.5s** 内整体**渐显**，与上面的退场**交叉重叠**进行 |

**锚点 = 店主头像**（唯一贯穿两页、不消失的元素）。

### 过渡 B：进入网页 → 进入作品

- 旧元素（进入网页，**含店主头像**）淡出，新元素（进入作品）渐显，**交叉重叠**
- **锚点 = 被点击的作品缩略图**：该缩略图**放大并移动到左边的大白框里 → 落位后直接开始播放静音 mp4**
- ⚠️ **头像不是锚点、不携带进作品页**，它随「进入网页」整页一起淡出

## RPG 绿色对话框（进入网页）

店主头像旁的绿色线框 = RPG 风格对话框，逐句推进台词。

**机制：**
- **打字机逐字显示**，节奏偏快，**单句打字动画 ≤ 1 秒**
- **非阻塞**：打字过程中**不锁浏览者其他操作**（可同时滚动 / 点缩略图 / 看 Main 区），对话框自顾自播
- 当前句打完 → 右下小三角（▽）开始**闪烁** = "继续/下一句"提示符
- 推进：点击 / 按键
- **skip 规则**：当前句未打完时点击 → 第一下**瞬间补全当前整句**，**再点一次**才跳下一句（RPG 标准，防误触）
- **台词文案待用户提供，先留空，把容器 + 打字机 + 推进状态机搭好**

## 音乐播放器彩蛋（左下角线框，跨页面常驻）

- 位置：左下角 L 形 / 层叠线框区域
- 功能：随时播放安安自己写的音乐，小彩蛋
- **UI 现阶段不设计** —— 线框本身就是入口/容器；后续会加类似「这是我写的音乐」的 UI 提示
- **默认静音、不自动播**（浏览器会拦自动播放），**用户点击线框才开始播**
- **跨页面持续存在**：页面过渡时音乐**不中断**，播放器状态（播放进度 / 播放-暂停）保留、不随过渡重置
  - 实现提示：播放器需挂在页面切换不卸载的层级（如 SPA 的常驻容器 / 持久 audio 元素），不要每页各自 new 一个

## 作品页导航（两条路径，行为不同）

作品页有两种离开方式，对应两种交互逻辑：

| 路径 | 入口 | 交互逻辑 | 去向 |
|------|------|---------|------|
| **返回**（自由挑） | 返回键 | **倒放进入动画**（过渡 B 反向）：当前作品的主展示 mp4 **缩回成缩略图、飞回它在网格里的原槽位**；同时随该页淡出的元素**向上升起重新出现**（遵循运动方向语法） | 回到「进入网页」作品总览网格，用户自己挑下一个感兴趣的 |
| **下一个**（看我安排） | 页面**最底部**点击 | **卷轴式过渡**：当前作品向上卷出，下一个作品卷入 | 按 `order` 递增进入下一个作品 |

**要点：**
- "返回"用倒放，所以飞回的目标槽位 = 当前作品 `id` 在网格中的位置（数据驱动，不能写死）
- `order` 是**全站一条线**（不分 3D/2D，从头串到尾），"下一个"沿这条线递增
- **到队尾时**：「下一个」按钮**变灰禁用（disabled）**，不循环、不跳转
- 两条路径都要保证**左下角音乐播放器不中断、状态保留**
- 卷轴过渡时，新作品的主展示 mp4 进入视口后再开始播，避免一次性预载所有作品视频

## 视频弹出放大（橙框细节视频，非原生全屏）

点击橙框里的 mp4 → 放大查看，但**不是浏览器原生全屏**：

- 视频从原位**放大弹出至屏幕约 70%**，**居中**显示
- 后方背景**变暗**，营造**正片叠底（multiply）**的压暗遮罩感
- **点击视频以外的空白处（遮罩区）** → 视频**缩回原位**、遮罩消失
- 开/合都是从橙框原位 scale 进出（与遮罩淡入淡出同步），不是硬切
- 实现：自定义 lightbox 容器（非 Fullscreen API），遮罩层 `mix-blend-mode: multiply` 或半透明黑

## 技术栈（拟定）

| 职责 | 技术 |
|------|------|
| 页面间过渡 | SPA 式单页 + 区块淡入淡出（保证音乐播放器常驻、头像锚点连续） |
| 作品渲染 | **数据驱动**：一份作品数据表渲染网格 + 详情页，新增作品只改数据 |
| 帧序列播放 | sprite sheet / `<img>` 序列切换 / `<canvas>`（按帧数与体积取舍） |
| 作品 mp4 | `<video>` autoplay + muted + loop + playsinline；橙框视频点击 → 自定义 lightbox 弹出放大（~70% 居中 + multiply 压暗遮罩），点遮罩缩回 |
| 模型 turntable | 帧序列 + 指针拖拽事件（pointermove）控制当前帧 + 闲置自动步进 |
| 返回（倒放） | 复用过渡 B 关键帧，方向取反：主 mp4 缩回缩略图飞回原网格槽位 |
| 下一个（卷轴） | 纵向 scroll/translateY 过渡，按 `order` 串联，视口内才播放新作品 mp4 |
| RPG 对话框 | 打字机状态机（typing / done / advancing）+ 非阻塞渲染 |
| 音乐播放器 | 常驻 `<audio>`，点击线框 toggle，状态跨页保留 |
| 缓动 | 关键帧查表 + smoothstep（复用 frin-site 方案） |
| blobworld 背景 | 渐变 + blob 素材（PNG/SVG），手绘装饰一律 PNG |
